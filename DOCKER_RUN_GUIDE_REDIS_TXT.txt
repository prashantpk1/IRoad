IROAD DOCKER QUICK START (DJANGO + REDIS)
=========================================

Purpose
-------
This guide is for a new developer who wants to run this project quickly.
It starts:
- `iroad-web` (Django app)
- `iroad-redis` (Redis)

App URL:
- http://localhost:8000

Internal Docker Redis URL:
- `redis://redis:6379`


Prerequisites (one-time)
------------------------
1) Install Docker Desktop and keep it running.
2) Make sure PostgreSQL is running on your Windows host machine.
3) In project root (`D:/iroad`), verify `.env` has:
   - `DB_NAME`
   - `DB_USER`
   - `DB_PASSWORD`
   - `DB_PORT` (usually 5432)
4) Open terminal in `D:/iroad`.

Important:
- `docker-compose.yml` already sets `DB_HOST=host.docker.internal`
  so the web container can connect to PostgreSQL on your host.


First run (recommended)
-----------------------
Run this once to build and start all services:

  docker compose up -d --build

Check containers:

  docker compose ps

You should see both `web` and `redis` in "Up" state.


Redis-only start (if web is already running locally)
----------------------------------------------------
If you only need Redis:

  docker compose up -d redis

Or if container already exists and is stopped:

  docker start iroad-redis


Verify Redis is healthy
-----------------------
1) Redis ping:

  docker exec -it iroad-redis redis-cli ping

Expected output:

  PONG

2) Optional Django helper check:

  docker compose exec web python manage.py shell -c "from superadmin.redis_helpers import redis_health_check; print(redis_health_check())"

Expected output:

  True


Daily use commands
------------------
Start services:

  docker compose up -d

Stop services:

  docker compose down

Restart services:

  docker compose restart

View logs:

  docker compose logs -f web
  docker compose logs -f redis

Rebuild web only:

  docker compose up -d --build web


After restart (PC or Docker Desktop)
------------------------------------
Because restart policy is `unless-stopped`, containers usually auto-start.
If not, run:

  docker compose up -d


Common issues
-------------
1) Port `6379` already in use
   (another Redis container/process is using it)

  docker stop iroad-redis
  docker rm iroad-redis
  docker compose up -d redis

2) Web cannot connect to database
   - Ensure PostgreSQL service is running on host.
   - Ensure PostgreSQL accepts TCP on port 5432.
   - Check `.env` DB credentials.

3) Redis error in app/login
   - Run `docker compose ps` and confirm redis is Up.
   - Run ping command and confirm `PONG`.
   - Restart web:

     docker compose restart web

4) Cloudflare tunnel not working
   - Point tunnel to `http://localhost:8000`
   - Confirm local app works first.
