Email & Password Settings

Configure email delivery, password policies, and two-factor authentication settings for your organization.

Email Configuration

Configure SMTP server settings used for sending transactional emails such as verification links and notifications.

SMTP Server Settings

The hostname or IP address of your SMTP server.
Common ports: 25 (SMTP), 465 (SSL), 587 (TLS/STARTTLS).
TLS is recommended for secure email delivery.
Name displayed as the email sender.
Email address used as the "From" address.
Email address where replies will be directed.

SMTP Authentication

Username for authenticating with the SMTP server.
Password or app-specific password for SMTP authentication.

Email Templates & Notifications

Enable or disable specific email notifications sent to users during account lifecycle events.

Account Lifecycle Emails

When enabled, new users must verify their email address before accessing the system.
A welcome email is sent after successful email verification.
Notify the user via email whenever their password is changed.
Notify the user when their account gets locked due to multiple failed login attempts.

Password Policy

Define password strength requirements and complexity rules enforced across all user accounts.

Length & Complexity

Minimum number of characters required. (6 – 128)
Maximum number of characters allowed. (16 – 128)
Password must contain at least one uppercase letter.
Password must contain at least one lowercase letter.
Password must contain at least one number.
Password must contain at least one special symbol.

Password History & Expiry

Number of previous passwords that cannot be reused. (0 = disabled)
Days before a password must be changed. (0 = never expires)
New users must change their temporary password on first login.

Password Reset Settings

Configure the reset link behavior and account lockout rules triggered by failed login attempts.

Reset Link Configuration

Duration before a password reset link expires. (5 – 1440 minutes)
How the password reset verification will be sent to the user.

Account Lockout Rules

Number of consecutive failed attempts before locking the account.
How long the account stays locked. (1 – 1440 minutes)
When enabled, accounts are automatically unlocked after the lockout duration expires. Otherwise, an admin must manually unlock.

Two-Factor Authentication (2FA)

Manage two-factor authentication options for enhanced login security.

2FA Configuration

When enabled, users will be required to provide a second verification factor during login.
Default verification method offered to users on login.
Duration before an OTP code expires. (60 – 900 seconds)
Let users select their preferred 2FA method from the available options.
Generate one-time backup codes that users can use if they lose access to their 2FA device.
