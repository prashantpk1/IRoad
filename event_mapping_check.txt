==================================================
5.3.3 Business Rules & Logic
==================================================

A) Event Immutability Rule:
   "System event list is hardcoded; Super Admin cannot create new event types from UI."
Status: YES
Reason:
- system_event is a fixed choices enum in model.
- UI form uses dropdown from those choices only.
- No endpoint/UI for dynamic event creation.

B) One mapping per event (architectural expectation)
Status: YES
Reason: system_event has unique=True in model; create view also blocks duplicates with explicit check.

C) Fallback Protocol:
   "If primary channel fails/timeout, instantly dispatch fallback template/channel."
Status: NO (for async path)
Reason:
- In dispatch_event_notification(), fallback is triggered only if _send() raises exception.
- For async sends, _send() queues Celery task and returns True immediately.
- Later delivery failures happen inside task, so main dispatcher does not catch them and cannot trigger fallback instantly.
- Fallback works for synchronous path (use_async_tasks=False), but not reliably for async production flow.
